# Quiz Game — План реализации

## Фаза 1: Фундамент
1. **Docker Compose** — postgres, backend, frontend, bot контейнеры.
2. **Go Backend каркас** — Gin-сервер, подключение к PostgreSQL через GORM, автомиграция моделей.
3. **Модели GORM** — Host, Quiz, Question, Option, Session, Participant, Answer.

## Фаза 2: Авторизация ведущего
4. **Регистрация / Логин** — bcrypt для паролей, JWT-токены.
5. **Middleware** — проверка JWT, извлечение host_id.

## Фаза 3: CRUD квизов
6. **Квизы** — создание, редактирование, удаление, список.
7. **Вопросы и варианты** — добавление/редактирование/удаление вопросов с вариантами (2-4, один правильный).

## Фаза 4: Сессии квиза
8. **Создание сессии** — генерация 6-значного кода, статус `waiting`.
9. **Управление сессией** — reveal (показать ответ), next (следующий вопрос), finish.
10. **WebSocket** — real-time обновления для экрана ведущего.

## Фаза 5: Участники и ответы
11. **Присоединение** — по коду сессии + telegram_id + ник.
12. **Отправка ответа** — валидация (сессия в статусе `question`, не отвечал ранее).
13. **Подсчёт очков** — 100 за правильный + бонус за скорость.

## Фаза 6: React Frontend
14. **Авторизация** — страницы логина/регистрации, хранение JWT.
15. **Личный кабинет** — список квизов, создание/редактирование.
16. **Экран сессии** — лобби (QR-код), игровой экран (вопросы, кнопки управления), таблица лидеров.
17. **WebSocket** — подписка на обновления сессии.

## Фаза 7: Telegram Bot
18. **Подключение к сессии** — /start → ввод кода → ввод ника.
19. **Игровой процесс** — получение вопросов, inline-кнопки ответов, отправка ответов через API.
20. **Результаты** — уведомление о правильности, итоговая таблица.

## Фаза 8: Финализация
21. **Таблица лидеров** — расчёт и отображение на фронте и в боте.
22. **Тестирование** — полный прогон сценария.
23. **.env.example, README** — инструкции по запуску.
